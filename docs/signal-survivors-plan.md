## 1. 게임 개요

### 1.1 컨셉

플레이어는 한 대의 차량. 사방에서 몰려오는 적 차량 무리를 자동공격과 4개의 스킬(QWER)로 처치한다.
적을 죽이면 경험치 오브가 드롭되고, 레벨업 시 3개 중 1개의 스킬을 선택한다.
같은 스킬을 중복 선택하면 강화되고, 특정 스킬 조합은 진화 스킬로 합쳐진다.

### 1.2 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| **시각적 쾌감 최우선** | 킬 이펙트, 파티클, 화면 흔들림, 쇼크링이 게임의 핵심 재미 |
| **의미 있는 스킬 조합** | 진화 스킬이 게임플레이를 근본적으로 바꿈 |
| **점진적 카오스** | 압도적이지만 관리 가능한 수준의 혼돈 |
| **따뜻한 미학** | 베이지/크림 톤을 유지하면서도 화려한 이펙트 |

## 2. 게임 구조

### 2.1 GameEngine 클래스 설계

```
GameEngine
├── app: PIXI.Application (720x720)
├── state: GameStateMachine
├── systems: System[]
│   ├── InputSystem          - 키보드/터치 입력 처리
│   ├── MovementSystem       - 플레이어/적 이동
│   ├── CollisionSystem      - 충돌 감지 (공간 해싱)
│   ├── SkillSystem          - 스킬 발동/쿨다운/진화
│   ├── DamageSystem         - 데미지 계산/적용
│   ├── SpawnSystem          - 적/웨이브 스폰
│   ├── XPSystem             - 경험치 오브/픽업/레벨업
│   ├── ParticleSystem       - 파티클 이펙트 풀링
│   ├── CameraSystem         - 화면 흔들림/줌
│   ├── ComboSystem          - 연속 킬 카운터/보너스
│   ├── UISystem             - HUD 렌더링
│   └── SoundSystem          - 효과음/BGM
├── pools: ObjectPool[]
│   ├── EnemyPool            - 적 오브젝트 풀 (최대 300)
│   ├── ProjectilePool       - 투사체 풀 (최대 500)
│   ├── ParticlePool         - 파티클 풀 (최대 2000)
│   └── XPOrbPool            - 경험치 오브 풀 (최대 500)
├── world: WorldState
│   ├── player: Player
│   ├── enemies: Enemy[]
│   ├── projectiles: Projectile[]
│   ├── xpOrbs: XPOrb[]
│   └── particles: Particle[]
└── config: GameConfig
```

### 2.2 상태 머신 (GameStateMachine)

```
States:
  LOADING     → 리소스 로드, 스프라이트시트 준비
  TITLE       → 타이틀 화면, "시작" 버튼
  PLAYING     → 메인 게임 루프 활성
  LEVEL_UP    → 스킬 선택 UI (게임 일시정지)
  BOSS_INTRO  → 보스 등장 연출 (1.5초)
  BOSS_KILL   → 보스 처치 연출 (2초)
  EVOLUTION   → 스킬 진화 연출 (2초)
  PAUSED      → 일시정지
  GAME_OVER   → 결과 화면
  VICTORY     → 30웨이브 클리어 시

Transitions:
  LOADING → TITLE           : 로딩 완료
  TITLE → PLAYING           : 시작 버튼
  PLAYING → LEVEL_UP        : 레벨업
  LEVEL_UP → PLAYING        : 스킬 선택 완료
  PLAYING → BOSS_INTRO      : 보스 웨이브 시작
  BOSS_INTRO → PLAYING      : 연출 종료
  PLAYING → BOSS_KILL       : 보스 처치
  BOSS_KILL → PLAYING       : 연출 종료
  PLAYING → EVOLUTION       : 진화 조건 달성
  EVOLUTION → PLAYING       : 연출 종료
  PLAYING → PAUSED          : ESC/터치
  PAUSED → PLAYING          : ESC/터치
  PLAYING → GAME_OVER       : HP ≤ 0
  PLAYING → VICTORY         : Wave 30 클리어
```

### 2.3 게임 루프 (60fps 타겟)

```
gameLoop(deltaTime):
  if state !== PLAYING: return

  1. InputSystem.update(dt)        // 입력 수집
  2. MovementSystem.update(dt)     // 위치 업데이트
  3. SpawnSystem.update(dt)        // 적 스폰
  4. SkillSystem.update(dt)        // 스킬 쿨다운/발동
  5. CollisionSystem.update(dt)    // 충돌 체크 (공간 해싱)
  6. DamageSystem.update(dt)       // 데미지 적용
  7. XPSystem.update(dt)           // XP 오브 이동/픽업
  8. ComboSystem.update(dt)        // 콤보 타이머/배수
  9. ParticleSystem.update(dt)     // 파티클 생명주기
  10. CameraSystem.update(dt)      // 화면 흔들림 감쇠
  11. UISystem.update(dt)          // HUD 갱신
  12. SoundSystem.update(dt)       // 사운드 큐 처리
```

### 2.4 공간 해싱 (Spatial Hashing)

```
셀 크기: 80x80 픽셀
그리드: 9x9 = 81 셀 (720/80)
각 프레임에서:
  1. 모든 적/투사체 위치로 그리드 갱신
  2. 투사체-적 충돌: 투사체가 속한 셀 + 인접 8셀만 검사
  3. 적-플레이어 충돌: 플레이어 주변 셀만 검사
  4. XP 오브 픽업: 플레이어 픽업 범위 내 셀만 검사
```

### 2.5 오브젝트 풀링

모든 게임 오브젝트는 풀에서 할당/반환. GC 압력 최소화.

```
ObjectPool<T>:
  pool: T[]           // 비활성 오브젝트
  active: Set<T>      // 활성 오브젝트
  factory: () => T    // 생성 함수
  reset: (T) => void  // 초기화 함수
  acquire(): T        // 풀에서 꺼내기
  release(T): void    // 풀로 반환
```

| 풀 | 최대 크기 | 초기 할당 |
|----|-----------|-----------|
| Enemy | 300 | 50 |
| Projectile | 500 | 100 |
| Particle | 2000 | 500 |
| XPOrb | 500 | 100 |
| DamageNumber | 200 | 50 |

## 3. 플레이어 시스템

### 3.1 플레이어 스프라이트

- 형태: 위에서 내려다본 차량 (탑다운 뷰)
- 크기: 32x48 픽셀 (가로x세로)
- 색상: #4A90D9 (파란색 차체) + #F5EDE0 (하이라이트)
- 이동 방향에 따라 차량 회전 (보간 적용, lerp 0.15)
- 피격 시 0.1초간 빨간색 틴트
- 무적 프레임 시 0.05초 간격 깜빡임

### 3.2 이동

```
입력 방식:
  - 키보드: WASD 또는 방향키
  - 터치/마우스: 조이스틱 (왼쪽 하단 가상 패드)
  - 드래그: 차량이 터치 방향으로 이동

이동 처리:
  1. 입력 벡터 정규화 (대각선 = 1.0 속도)
  2. 속도 적용: position += direction * speed * dt
  3. 월드 경계 클램프 (0~720)
  4. 차량 회전: angle = lerp(currentAngle, targetAngle, 0.15)
```

### 3.3 기본 스탯

| 스탯 | 기본값 | 최대값 | 설명 |
|------|--------|--------|------|
| HP | 100 | 300 | 체력 |
| 이동 속도 | 180 px/s | 280 px/s | 초당 이동 픽셀 |
| 픽업 범위 | 60 px | 180 px | XP 오브 자석 범위 |
| 무적 시간 | 0.5초 | 1.2초 | 피격 후 무적 프레임 |
| 기본 공격력 | 10 | - | 스킬 배율의 기준 |
| 쿨다운 감소 | 0% | 40% | 스킬 쿨다운 감소율 |
| 범위 증가 | 0% | 50% | 스킬 범위 증가율 |
| 투사체 추가 | 0 | +3 | 투사체 스킬에 추가 발사체 |

### 3.4 자동 공격 (Auto Attack)

- 가장 가까운 적 방향으로 자동 발사
- 쿨다운: 0.8초 (쿨감 적용)
- 데미지: 기본 공격력 x 1.0
- 투사체 속도: 400 px/s
- 비주얼: 작은 원형 탄환 (#4A90D9, 8px)
- 적중 시 작은 스파크 이펙트

### 3.5 무적 프레임 (I-Frame)

```
피격 처리:
  1. 데미지 적용
  2. 무적 상태 ON (0.5초 기본)
  3. 넉백: 피격 방향 반대로 50px (0.15초)
  4. 화면 흔들림: intensity 3, duration 0.15초
  5. 피격 파티클: 빨간 스파크 8개
  6. 깜빡임 애니메이션 시작 (alpha 0.3 ↔ 1.0)
  7. 무적 시간 종료 후 정상 복귀
```

### 3.6 체력 회복

- 자연 회복 없음
- 웨이브 클리어 시 최대 HP의 10% 회복
- 보스 처치 시 최대 HP의 25% 회복
- 특정 스킬 진화 시 흡혈 효과 부여 가능

## 4. 스킬 시스템

### 4.1 구조

```
SkillSlot:
  key: 'Q' | 'W' | 'E' | 'R'
  skill: Skill | null
  level: 1~5
  cooldownRemaining: number
  isEvolved: boolean

Skill:
  id: string
  name: string
  description: string
  icon: string (이모지 또는 심볼)
  type: 'projectile' | 'area' | 'orbital' | 'passive' | 'summon'
  baseCooldown: number
  baseDamage: number (공격력 배율)
  baseRange: number
  color: string
  levels: SkillLevel[5]
  evolutionPair: string | null  // 합쳐지는 상대 스킬 ID
  evolvedSkill: string | null   // 진화 결과 스킬 ID
```

### 4.2 스킬 획득/강화 규칙

```
레벨업 시:
  1. 현재 보유하지 않은 스킬 풀에서 랜덤 2개 선택
  2. 현재 보유 중이며 Lv5 미만인 스킬에서 1개 선택
  3. 3개의 선택지 제공
  4. 빈 슬롯이 없으면 새 스킬 대신 강화 옵션만 제공
  
진화 체크:
  - 스킬 A와 스킬 B 모두 Lv3 이상
  - 둘 다 장착 중
  - → 다음 레벨업 시 진화 선택지가 최우선으로 등장
  - 진화 선택 시 두 스킬이 하나의 진화 스킬로 합쳐짐
  - 진화 스킬은 항상 Lv1에서 시작하며, Lv3까지 강화 가능
  - 빈 슬롯 1개가 확보됨
```

### 6.3 쇼크링 (Shock Ring)

```
ShockRing:
  x, y: number         // 중심
  radius: number        // 현재 반경
  maxRadius: number     // 최대 반경
  duration: number      // 총 지속 시간
  elapsed: number       // 경과 시간
  color: number         // 외곽선 색상
  lineWidth: number     // 외곽선 두께
  alpha: number         // 투명도 (시간에 따라 감소)

렌더링:
  progress = elapsed / duration
  radius = maxRadius * easeOutQuad(progress)
  alpha = 1.0 - progress
  lineWidth = initialWidth * (1.0 - progress * 0.5)
  → PIXI.Graphics로 원 그리기
```

### 6.4 화면 흔들림 (Screen Shake)

```
CameraShake:
  intensity: number     // 흔들림 강도 (px)
  duration: number      // 지속 시간
  decay: number         // 감쇠율 (보통 0.9)
  
  매 프레임:
    offsetX = (Math.random() * 2 - 1) * intensity
    offsetY = (Math.random() * 2 - 1) * intensity
    intensity *= decay
    stage.position.set(offsetX, offsetY)
  
  duration 경과 후 stage.position.set(0, 0) 복귀
```

**초신성 폭발**: 골드 쇼크링 확산 + 화이트 플래시 + 금빛 파티클 100개
**오버드라이브**: 골드 오라 + 속도 잔상 (이전 위치에 반투명 복사본 3개)
