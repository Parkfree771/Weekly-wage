# Signal Survivors — 공격 이펙트 개발서

---

## 황금률

1. **절대 대충하지 않는다.** 공격 이펙트는 이 게임의 거의 모든 것이다. 에이전트한테 대충 시키는 일은 있어서는 안 된다.
2. **하나하나 장인이 도자기를 빚듯이 한땀한땀 코딩한다.**
3. **우리는 최고의 게임 개발자다.**
4. **이 개발서가 헌법이다.** 개발서에 정의된 내용(기술 스택, 아키텍처, 원소 정의, 조합 시스템, 개발 순서 등)에 어긋나는 명령이 들어오면 무조건 반론을 제기한다. 개발서를 먼저 수정하고 나서 코드를 수정한다. 절대 개발서를 무시하고 코딩하지 않는다.

## 기술 스택

| 항목 | 선택 | 비고 |
|------|------|------|
| 게임 엔진 | PixiJS v7.4.3 (유지) | WebGL 2D 렌더러, 기존 게임 코드 그대로 |
| 이펙트 (기본) | @pixi/particle-emitter | 파티클 기반 이펙트. JSON 설정 + WebGL 배치 렌더링. 수천 개 파티클도 60fps |
| 이펙트 (특수) | 커스텀 GLSL 셰이더 | 블랙홀, 중력 왜곡 등 특수 연출에만 사용. 후순위 |
| 언어 | TypeScript | 기존과 동일 |
| 빌드 | Next.js | 기존과 동일 |

### 파티클 이미터 사용 방식

```typescript
import { Emitter } from '@pixi/particle-emitter';

// 1. 이미터 생성 (컨테이너 + JSON 설정)
const emitter = new Emitter(effectContainer, {
  lifetime: { min: 0.3, max: 0.8 },
  frequency: 0.01,
  emitterLifetime: -1, // 무한
  pos: { x: 0, y: 0 },
  behaviors: [
    { type: 'alpha', config: { alpha: { list: [{ time: 0, value: 0.8 }, { time: 1, value: 0 }] } } },
    { type: 'scale', config: { scale: { list: [{ time: 0, value: 0.5 }, { time: 1, value: 0.1 }] } } },
    { type: 'color', config: { color: { list: [{ time: 0, value: '3b82f6' }, { time: 1, value: '1d4ed8' }] } } },
    { type: 'moveSpeed', config: { speed: { list: [{ time: 0, value: 200 }, { time: 1, value: 50 }] } } },
    { type: 'spawnShape', config: { type: 'torus', data: { x: 0, y: 0, radius: 80, innerRadius: 0 } } },
  ],
});

// 2. 매 프레임 업데이트
ticker.add((dt) => {
  emitter.update(dt / 60);
  emitter.updateOwnerPos(player.x, player.y);
});
```

### 이펙트 아키텍처

```
app/minigame/survivors/
├── effects/                    ← 신규
│   ├── EffectManager.ts        ← 이펙트 생성/파괴/업데이트 관리
│   ├── WaterEffect.ts          ← 물 속성 이펙트
│   ├── EarthEffect.ts          ← 흙 속성 이펙트
│   └── ...
├── renderer.ts                 ← 기존 (generic 렌더링 + 이펙트 연결)
├── engine.ts                   ← 기존 (게임 루프)
├── weapons.ts                  ← 무기 정의 (새로 채울 예정)
└── ...
```

> 특수 연출(블랙홀 왜곡, 중력 렌즈 등)이 필요할 때만 shaders/ 폴더를 추가하여 커스텀 GLSL을 사용한다.

### 핵심 원칙

- **게임 로직과 이펙트 렌더링을 완전 분리한다.** engine.ts/weapons.ts는 "무엇이 발동됐는지"만 알려주고, 어떻게 보여줄지는 effects/ 폴더가 전담한다.
- **각 속성의 이펙트는 독립된 클래스 + 독립된 셰이더 파일로 관리한다.** 서로 영향을 주지 않는다.
- **기본 이펙트는 PIXI.Graphics + @pixi/particle-emitter 하이브리드로 만든다.** 파동/링 같은 연속 도형은 Graphics로 직접 그리고, 물보라/불꽃 같은 개별 파티클은 emitter를 쓴다. 커스텀 GLSL 셰이더는 블랙홀/중력 왜곡 같은 특수 연출에만 사용한다.

## 이펙트 구현 규칙 (삽질 방지)

### 1. generic weaponEffect와 커스텀 이펙트를 절대 동시에 쓰지 않는다

- **문제**: 커스텀 이펙트(EffectManager)를 만들면서 기존 `spawnAutoEffect()`도 함께 호출하면, renderer.ts의 generic 렌더러가 따로 그림을 그려서 **흰색 잔상이 겹쳐 보인다.**
- **해결**: 커스텀 이펙트를 만든 속성은 `autoAttack()`에서 `spawnAutoEffect()` 호출을 완전히 제거한다. 데미지 판정도 EffectManager 쪽에서 처리하거나 별도로 분리한다.
- **renderer.ts 방어 코드**: 혹시 모를 상황 대비, 커스텀 이펙트의 uniqueId는 renderer.ts 상단에서 `g.visible = false; continue;`로 스킵한다.

### 2. 이펙트 위치 갱신은 반드시 gameLoop에서 매 프레임 한다

- **문제**: `autoAttack()` 안에서 위치를 갱신하면, autoAttack의 쿨다운 타이머(AUTO_ATTACK_INTERVAL = 30프레임) 때문에 **30프레임에 한 번만 위치가 업데이트**되어 뚝뚝 끊긴다.
- **해결**: `updateElementEffects()` 메서드를 만들어 gameLoop에서 매 프레임 호출한다. 이 메서드가 슬롯의 원소를 체크하고, 해당 이펙트의 `setPosition(player.x, player.y)`를 매 프레임 갱신한다.
- **구조**: `autoAttack()`은 데미지/쿨다운만 담당, `updateElementEffects()`는 이펙트 시각만 담당. 완전 분리.

### 3. GLSL Filter를 월드 공간 이펙트에 쓰지 않는다

- **문제**: `PIXI.Filter`는 화면 후처리(post-processing)용이다. 투명 스프라이트에 Filter를 붙여서 월드 공간 이펙트를 만들면 **좌표가 안 맞고, 텍스처가 비어서 렌더링이 깨진다.**
- **해결**: 월드 공간에 직접 그리는 이펙트는 `PIXI.Graphics` 또는 `PIXI.Mesh`를 사용한다. Filter는 화면 전체에 적용하는 후처리(블룸, 색수차 등)에만 쓴다.

## 6원소 정의

### 색상 팔레트

### 물 (Water)

- **공격 유형**: 장판형
- **컨셉**: 파도
- **단일 효과**: 캐릭터 주변으로 물결 파동이 동심원으로 퍼져나간다. 파동에 닿은 적은 밀려나고(넉백) 데미지를 받는다.
- **핵심 시각 요소**:
  - 중심에서 바깥으로 퍼지는 반투명 물결 링
  - 파동이 지나간 자리에 잔물결 잔상
  - 물보라 파티클 (백색 스파크)
  - 파동 전면부가 가장 밝고, 뒤로 갈수록 투명해짐
- **게임 효과**: 넉백 + 데미지
- **셰이더 키워드**: 동심원 SDF, 사인파 왜곡, 프레넬 반사광

---

### 흙 (Earth)

- **공격 유형**: 장판형
- **컨셉**: 사막
- **단일 효과**: 캐릭터 주변에 사막 모래 필드가 깔린다. 필드 안에 들어온 적은 모래에 발이 빠지듯 느려지고(슬로우), 지속적으로 데미지를 받는다.
- **핵심 시각 요소**:
  - 바깥쪽 진한 갈색 → 안쪽 밝은 모래색 그라데이션
  - 안쪽으로 빨려드는 유사(流砂) 나선 패턴 → 둔화 시각 표현
  - 모래 파문이 중심에서 바깥으로 펄스
  - 소형 샌드 데빌(모래 회오리)이 필드 내부를 공전
  - 경계에서 모래 먼지 파티클
- **게임 효과**: 슬로우 + 지속 데미지
- **셰이더 키워드**: 노이즈 텍스처, 나선 UV 왜곡, 열기 아지랑이 디스토션

---

### 불 (Fire)

- **공격 유형**: 방사형
- **컨셉**: 화염방사기
- **단일 효과**: 캐릭터의 이동 방향(또는 가장 가까운 적 방향)으로 부채꼴 화염이 계속 발사된다. 불길에 닿은 적은 타오른다.
- **핵심 시각 요소**:
  - 부채꼴 콘 형태의 화염 스트림
  - 뿌리 부분은 밝은 백색/금색, 끝으로 갈수록 주홍→어두운 적색
  - 불꽃 파티클이 바깥쪽으로 흩날림
  - 화염 가장자리에 열기 왜곡(heat distortion)
  - 바닥에 그을음 잔상
- **게임 효과**: 지속 데미지 (틱 데미지)
- **셰이더 키워드**: 펄린 노이즈 화염, 부채꼴 마스킹, 열기 디스토션

---

### 빛 (Light)

- **공격 유형**: 투사형
- **컨셉**: 키자루 (원피스) — 빛의 속도 레이저
- **단일 효과**: 작은 광자 알갱이들이 사방에서 캐릭터 쪽으로 천천히 모인다(차징). 충분히 모이면 캐릭터의 머리 방향에서 강렬한 레이저 빔이 일직선으로 발사된다. 빔이 지나간 자리는 잔광이 남는다.
- **핵심 시각 요소**:
  - 차징 단계: 주변에서 중심으로 모이는 미세한 광점들, 중심부 코어가 점점 밝아짐
  - 발사 단계: 가늘고 강렬한 백색 코어 + 바깥쪽 금색 글로우, 빔 주변 렌즈 플레어
  - 빔 경로에 빛 잔상 (서서히 소멸)
  - 발사 순간 화면 플래시
- **게임 효과**: 관통 고데미지 (직선 범위)
- **셰이더 키워드**: 블룸, 가우시안 글로우, 직선 SDF, 렌즈 플레어

---

### 전기 (Electric)

- **공격 유형**: 투사형
- **컨셉**: 감전 체인
- **단일 효과**: 캐릭터에서 가장 가까운 적에게 전기가 발사된다. 적에게 적중하면 "지지직" 감전 상태가 되고, 근처 다른 적에게 전기가 연쇄 전이된다.
- **핵심 시각 요소**:
  - 지그재그 번개 볼트 (매 프레임 경로가 미세하게 변함 → 지직거림)
  - 번개 코어는 순백, 바깥 글로우는 보라/라벤더
  - 적중 지점에 전기 스파크 폭발
  - 연쇄 시 적과 적 사이를 잇는 아크
  - 감전된 적 주변에 작은 전기 파티클이 지지직 튐
- **게임 효과**: 데미지 + 연쇄 전이 (체인 라이트닝)
- **셰이더 키워드**: 지그재그 라인 생성, 프랙탈 분기, 글로우 블룸

---

### 암흑 (Dark)

- **공격 유형**: 장판형
- **컨셉**: 중력장
- **단일 효과**: 캐릭터 주변에 중력장이 형성된다. 영역 안의 적은 중심으로 천천히 끌려오며(흡인), 지속적으로 데미지를 받는다. 물(넉백)의 정반대.
- **핵심 시각 요소**:
  - 공간이 왜곡되는 중력 렌즈 효과 (배경이 휘어짐)
  - 바깥에서 중심으로 빨려드는 어두운 입자 흐름
  - 중심부에 짙은 보라/검정 코어
  - 영역 경계에 빛이 굴절되는 링 (사건의 지평선 느낌)
  - 끌려오는 적 주변에 공간 왜곡 잔상
- **게임 효과**: 흡인 + 지속 데미지
- **셰이더 키워드**: 중력 렌즈 디스토션, UV 와프, 보로노이 노이즈, 사건의 지평선 링
- **우주 발전 방향**: 암흑 = 중력. 조합 시 블랙홀, 초신성, 빅뱅 등 우주 현상으로 자연 확장.

| 조합 | 우주 현상 | 방향 |
|------|-----------|------|
| 암흑×3 | 블랙홀 | 중력장 극대화, 모든 것을 삼킴 |
| 암흑+빛 | 초신성 / 일식 | 중력 붕괴 + 빛의 폭발 |
| 암흑+불 | 항성 붕괴 / 흑염 | 중력에 짓눌린 불 |
| 암흑+전기 | 자기장 폭풍 | 중력 + 전자기력 = 우주 방사선 |
| 암흑+빛+불 | 빅뱅 | 중력+빛+에너지 = 우주 탄생 |

## 조합 시스템

### 원칙

- 슬롯은 3칸. 원소 오브를 하나씩 채운다.
- **이펙트를 결정하는 것은 "고유 원소의 종류"**이다. 순서 무관, 중복은 스케일링.
- 같은 원소가 중복되면 이펙트 디자인은 동일하고, **데미지/크기/범위만 강화**된다.

### 단계 구조

| 단계 | 채운 슬롯 | 이펙트 결정 기준 | 설명 |
|------|-----------|-----------------|------|
| 1단계 | 1칸 | 단일 속성 | 기본 자동공격 이펙트 |
| 2단계 | 2칸 | 고유 원소 종류 수 | 같은 원소 ×2 → 1단계 강화 / 다른 원소 ×2 → 2속성 조합 이펙트 |
| 3단계 | 3칸 | 고유 원소 종류 수 | 아래 표 참고 |

### 3단계(슬롯 3칸) 세부

| 조합 형태 | 예시 | 이펙트 | 스케일 |
|-----------|------|--------|--------|
| A+A+A | 불+불+불 | 1단계 "불" 이펙트 | 최종 강화 (3단계) |
| A+A+B | 불+불+물 | 2단계 "불+물" 조합 이펙트 | 강화 (3단계) |
| A+B+C | 불+물+흙 | 3속성 고유 이펙트 | 기본 (3단계) |

### 설계해야 할 고유 이펙트

| 분류 | 개수 | 산출 |
|------|------|------|
| 단일 속성 | 6개 | 6원소 각 1개 |
| 2속성 조합 | 15개 | C(6,2) = 15 |
| 3속성 조합 | 20개 | C(6,3) = 20 |
| **합계** | **41개** | 고유 이펙트 디자인 |

나머지 15개(총 56조합 - 41)는 기존 이펙트에 스케일링만 적용.

### 스케일링 규칙

같은 이펙트의 단계가 올라갈 때 변하는 것:

| 항목 | 1단계 → 2단계 | 2단계 → 3단계 |
|------|---------------|---------------|
| 데미지 | ×1.0 | ×1.5 |
| 범위/크기 | ×1.0 | ×1.3 |
| 시각 밀도 | 기본 | 파티클 증가, 글로우 강화 |

> 구체적인 수치는 밸런싱 단계에서 조정. 위 수치는 초안.

### 개발 순서

**Phase 1 — 단일 속성 (6개)**
1. ✅ 물 (파도 장판) — 완료
2. 흙 (사막 장판)
3. 불 (화염방사기)
4. 빛 (레이저 빔)

## 구현 완료 이펙트 상세

### 물 (Water) — 1단계 완료

**파일**: `effects/WaterEffect.ts`, `effects/EffectManager.ts`

**구현 방식**: PIXI.Graphics (일렁이는 링) + PIXI.Graphics (물보라 파티클)

**시각 구성**:
- 동심원 파동 링 3~4겹이 중심에서 바깥으로 확장 (28프레임 간격 생성)
- 각 링은 사인파 3겹(주파수 5, 8, 13) 중첩으로 일렁거림 — 단순 원이 아닌 물결 형태
- 파봉(crest)은 두껍고 밝은 파랑, 파곡(trough)은 얇고 어두운 남색
- 파봉 꼭대기에 강조 점
- 링 뒤에 ADD 블렌드 글로우
- 물보라 파티클이 파동 전면부에서 바깥으로 분출

**색상 팔레트** (흰색 없음, 파란 계열 통일):
- 파봉: `0x2563eb`
- 파곡: `0x1d4ed8`
- 글로우: `0x1e40af`
- 물보라 코어: `0x2563eb`
- 물보라 글로우: `0x1e40af`
- 파봉 강조: `0x3b82f6`

**게임 효과**:
- 넉백: 매 프레임 범위 내 적을 바깥으로 밀어냄 (중심에 가까울수록 강함, 최대 2.5px/frame)
- 데미지: 0.5초마다 8 데미지

**엔진 연결**:
- `updateElementEffects()` — gameLoop에서 매 프레임 호출, 슬롯에 물 원소 있으면 이펙트 활성 + 위치 추적 + 넉백/데미지 처리
- `autoAttack()` 물 case — 비어있음 (updateElementEffects가 전담)
- `renderer.ts` — `auto_water` uniqueId 스킵 처리
